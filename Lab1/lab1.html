Investigacion


URL: (Uniform Resource Locator) Es la direccion unica que que tiene cada uno de los elementos que se encuentran en internet, ya sea una pagina,un video o una imagen; que nos permite localizar y acceder a dicho elemento utilzando su direccion[1].


MIME: (Multipurpose Internet Mail Extensions) Es un estandar para expandir las capacidades limitadas del correo electronico
para permitir entre otras cosas la insercion de documentos como imagenes sonido y texto en los mensajes. Ademas permitio aumentar la cantidad
de archivos a adjuntar, utilizar caracteres que no pertenecen al codigo ASCII, utilizar diferentes fuentes, colores,tamanios y estilos y
escribir texto ilimitado[2].


Metodos y codigos de errores HTTP:


Métodos

  • GET: Pide la representacion de un recurso especifico, solo recupera datos.
  • HEAD: Pide la representacion de un recurso especifico,pero sin el cuerpo de la respuesta.
  • POST: Envia un elemento a un recurso especifico.
  • PUT: Reemplaza las representaciones actuales del recurso destinatario con la carga util de la peticion.
  • DELETE: Borra un elemento en especifico.
  • CONNECT: Conecta con el servidor identificado por el recurso.
  • OPTIONS: Muestra las opciones de comunicacion del elemento destinatario.
  • TRACE: Realiza una prueba de bucle de retorno de mensaje a lo largo de la ruta del elemento destinatario.
  • PATCH: Aplica modificaciones parciales a un elemento.[3]


    • Códigos de error

  • 1xx: Respuestas informativas.
  • 2xx: Peticiones Correctas.
  • 3xx: Redirecciones.
  • 4xx: Errores del cliente.
  • 5xx: Errores del servidor


    • XML:(Extensive Markup Language) Es un formato general para documentos organizados en internet. Permite intercambiar datos
    estructurados en distintas plataformas y es muy utilizado en hojas de calculo, bases de datos y editores de texto[4].


    XHTML:(eXtensible Hypertext Markup Language) Es un lenguaje parecido a HTML, con algunas diferencias que lo hacen mas complejo y utilizable para modelar paginas web. Se considera la version de XML de HTML, teniendo la misma rigidez y normas de XML. Lo cual produce ventajas como procesar contenido por cualquier porgrama informatico y ayudar a los navegadores a no tener que interpretar lo que el desarrollador ha escrito o sulucionar errores de codigo[5].


    HTML5: Es la ultima version de HTML, que contiene nuevos elementos herramientas y tecnologias que le permite tener a las paginas web mayor diversidad. Da mas precision en su contenido, permite almacenar datos localmente sin conexión, da exclente soporte a contenido multimedia Nuevos caracteristicas graficas 2D y 3D, mejor rendimiento y uso de hardware API's para la entrada y salida del dispositivo y CSS3[6].


    JS: (Javascript) Es un lenguaje de programacion utilizado para el desarrollo y diseño web. No requiere compilacion, ya que el encargado de interpretar el codigo es el navegador[7].


    PHP: (Hypertext Preprocessor) Es un lenguaje de programacion abierto adecuado para el desarrollo web, el cual puede ser incrustado en HTML. Esta enfocado principalmente en la programacion de scripts del lado del servidor[8].


    JSP: (Java Server Pages) Es una tecnolgia orientada a la creacion de paginas web con Java. Con este se pueden crear aplicaciones web
    en diferentes servidores web y plataformas. Las paginas JSP estan compuestas de codigo HTML/XML y etiquetas especiales para poder porgramar con la sintaxis de Java [9].


    Ajax: (Asynchronus JavaScript And XML) Es una tecnica que permite leer datos de un servidor web despues de que se cargo, actualizar una pagina sin volverla a cargar y enviar de fondo datos a un servidor web[10].

    Ciclo de vida y de desarrollo de los sistemas de informacion

    Un sistema de informacion a lo largo de su desarrollo pasa por varios pasos o etapas. Estas son: Planificacion, Analisis, disenyo, implementacion, pruebas, instalacion o despliegue y uso y mantenimiento. A lo largo de este trabajo se explicara a detalle cada una de estas etapas en el orden previamente establecido, ademas de ejemplos para tener un mejor entendimeinto de cada etapa.

    Ciclo de desarrollo de un sistema:

    Planificacion: En esta etapa se determinan caracteristicas importantes como el ambito del proyecto, se have un estudio de viabilidad, se analizan los riesgos relacionados con el proyecto, el costo que tendra, generar un plan de trabajo y un itinerario y asignar recursos a las fases del proyecto. Por delimitar un proyecto, se entiende que es establecer las situaciones o cuestiones que se resolveran por medio del proyecto y cuales no se van a incluir dentro del mismo, una vez determinada esta informacion se escribe un documento especificando los requisitos funcionales del proyecto y forma parte del contrato que se firma con el cliente. Esta fase es muy importante, ya que determina en gran medida que se finalice excitosamente el proyecto.[15]

    Analisis: Durante esta etapa se busca lo que el sistema debe hacer, es un proceso que busca lo que se necesita y tener una buena comprension de sus requerimientos. Es muy importante verificar en esta etapa verificar lo que el cliente realmente quiere y tener la mayor informacion posible para no crear un sistema al cual se tenga que hacer modificaciones costosas mas tarde, siendo mas barato y sencillo que en etapas mas avanzadas. Algunas buenas prácticas para esta etapa son: Una buena comunicacion con las personas involucradas por medio de tecnicas de entrevistas, cuestionarios o prototipos y la realizacion de modelos que ayuden a comunicar la estructura del sistema, especificar su comportamientos y descubrir oportunidades de mejora o simplificacion.[15]

    Disenio: En esta fase se revisan las posibles alternativas de como implementar el sistema de informacion y se debe de decidir el disenio arquitectonico y de estructura que tendra el sistema. Esta etapa es compleja y se realiza de forma iterativa para poder refinar la solucion propuesta. Esta etapa presenta dos facetas: el disenio de la base de datos y el disenio de las aplicaciones y la interfaz que le permitan al usuario hacer uso del sistema de informacion.[15]

    Implementacion: En esta etapa comienza la codificacion del proyecto. Se han de seleccionar las herramientas adecuadas, un entorno de desarrollo y un lenguaje de programacion apropiados para el sistema que se va a construir. El codigo debe de ser legible y tener estructuras de control bien estructuradas. Tambien es necesario seleccionar buenos algoritmos y tener estructuras de datos y una logica adecuada para mantener la aplicacion lo mas sencillo posible. Ademas el tener comentarios, sangrias y separaciones de bloques dentro del codigo facilita la interpretacion de nuestro codigo.[15]

    Pruebas: Esta etapa se enfoca en detectar los errores que se pudieron haber cometido en las fases anteriores del proyecto y corregirlos, para que el usuario no sufra con ellos. Entre algunos tipos de prueba existen las pruebas de unidad que comprueban que un componente del sistema funcione correctamente; las pruebas de integracion que verifican errores en las interfaces al juntar varios componentes; las pruebas alfa que ayudan pulir la interfaz del sistema desde un punto de vista de un usuario al sistema; las pruebas beta que se realizan con usuarios ajenos al grupo de desarrollo y que verifican asuntos vitales para que se el sistema exitoso en el mercado y test de aceptacion donde se acepta el producto y se finaliza su desarrollo.[15]

    Instalacion y depliegue: En esta etapa se pone el sistema en funcionamiento. Se debe de planificar el entorno de funcionamiento del sistema, revisando el hardware y software necesarios, la configuracion fisisca, las redes de interconexion de equipos, el acceso a sistemas externos, el sistema operativo, entre otros. [15]

    Mantenimiento: Consiste en eliminar los defectos detectados en su vida util, adaptarlo a las nuevas necesidades del usuario, de los sistemas operativos o diferentes hardware y a darle nuevas funciones que son desables y que mejoren el sistema ya existente.[15]

    Ciclo de vida de un sistema

    Investigacion preeliminar: Hacer una busqueda de los problemas que nuestro sistema busque resolver y en realizar la planificacion del prouecto.[16]

    Determinacion de requerimientos: Consiste en averiguar que es lo que se necesita y se va a requerir que haga el sistema.[16]

    Disenio:Se estudian la posibles implementaciones del sistema. Se seleccionan las herramientas adecuadas para el trabajo y el lenguaje de programacion.[16]

    Pruebas: Se detectan y corrigen los errores de las etapas anteriores del proyecto.[16]

    Implementacion y mantenimiento: Se planifica el entorno en el que el sistema va a funcionar y sus configuraciones de hardware y software. Tambien se eliminan los defectos del sistema, se adapta a nuevas necesidades y se la mejoras al sistema ya existente.[16]

    En conclusion se puede decir que el ciclo de desarrollo y el ciclo de vida al producir un sistema de un software son fundamentales para el desarrollo y la finalizacion exitosa de un proyecto. Seguir los pasos en el orden en el que se mencionan es de suma importancia, ya que como se pudo ver en la etapa de analisis y disenio, es muy importante tener claro las necesidades del cliente y que es lo que el sistema debe hacer para resolver estas necesidades. Una vez hecho esto definir y planificar la estructura y el disenio para el sistema de informacion. El no tener una buena planeacion del proyecto implica tener que rehacer grandes partes del proyecto y gastar muchos recursos en hacer cambios y adaptaciones, por lo que la planeacion y el analisis resultan partes fundamentales del proyecto, al igual que todas las etapas de estos ciclos.
