Cuestionario

1. ¿Por qué es importante hacer un session_unset() y luego un session_destroy()?
Para liberar el valor de las variables globales de sesión y no se queden guardadas después de que se destruya toda la información de la sesion



2. ¿Cuál es la diferencia entre una variable de sesión y una cookie?
Las cookies guaran datos a nivel local en el navegador del usuario y las sesiones alamacenan los datos en el servidor web.



3. ¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?
Guarda el nombre de la fotografía, además del la fecha, hora, minuto y segundo, lo cual hace que su información sea única.



4.¿Qué es CSRF y cómo puede prevenirse?
Cross-site request forgery es un tipo de explotación de una página web, dónde comandos no autorizados a un usuario de confianza de la página web. Se puede prevenir con autentificación adicional en las solicitudes que permiten a la aplicación web detectar peticiones de localidades no autorizadas.
